## SSH key setup for automated deploy (manual steps)

These steps must be done once by hand. The GitHub Actions workflow at
`.github/workflows/deploy.yml` expects two repository secrets.

### 1. Generate a dedicated deploy key pair

```bash
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/schneebrenner_deploy -N ""
```

This creates:
- `~/.ssh/schneebrenner_deploy` ‚Äî private key (goes to GitHub Secret)
- `~/.ssh/schneebrenner_deploy.pub` ‚Äî public key (goes to the server)

### 2. Add the public key to the hosting server

```bash
ssh -p 2244 websezrvz_3hx63t@schneebrenner.de \
  "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys" \
  < ~/.ssh/schneebrenner_deploy.pub
```

### 3. Get the server's known_hosts entry

```bash
ssh-keyscan -p 2244 schneebrenner.de
```

Copy the output ‚Äî you'll need it in the next step.

### 4. Add secrets to the GitHub repository

Go to: https://github.com/klasch5town/schneebrenner_de/settings/secrets/actions

Add these two secrets:

| Secret name | Value |
|---|---|
| `DEPLOY_SSH_KEY` | Contents of `~/.ssh/schneebrenner_deploy` (private key, including the `-----BEGIN...` lines) |
| `DEPLOY_SSH_KNOWN_HOSTS` | Output of `ssh-keyscan -p 2244 schneebrenner.de` |

### 5. Verify

Push a commit to `main` and check the Actions tab on GitHub.
